Integração CRM Seller (Notas Fiscais)

Visão Geral​

Sistema de integração responsável por enviar notas fiscais do Protheus para o sistema CRM Seller via API REST, incluindo dados de vendas, devoluções e itens detalhados.

Funcionalidades Principais​

  • Envio automático de notas fiscais (vendas e devoluções)
  • Processamento por filial com isolamento de dados
  • Controle de duplicação via tabela ZRM
  • Processamento assíncrono para melhor performance
  • Logs detalhados para monitoramento e debug
  • Finalização automática pós-integração

Arquitetura do Sistema​

Estrutura de Arquivos​

ag/2026/V3.0/integracao_clientes/
├── agcpdcrm.prw              # Fonte principal
├── crmCredencial.prw         # Gerenciamento de credenciais
├── crmintegracao.prw         # Integrações grupo/marca/modelo
└── CRM_Seller_Documentacao.md # Esta documentação

Dependências​

  • crmCredencial.prw: Configurações e URLs da API
  • Tabela ZRM: Controle de integração e logs
  • Parâmetros: ESSELLULT*, ES_EMPRCRM, ES_FILCRM

Funções Principais​

1. U_CRMSELLER(aParam, cTipo, cDataIni, cDataFim)​

Função principal para integração de notas fiscais.

Parâmetros​

ParâmetroTipoObrigatórioDescriçãoExemplo
aParamArray{empresa, filial, tipo}{"01","01","PRIMEIRA"}
cTipoString[OBSOLETO] Mantido por compatibilidade""
cDataIniStringData inicial (YYYYMMDD)"20260301"
cDataFimStringData final (YYYYMMDD)"20260310"

Exemplos de Uso​

// Carga primeira - todas as notas pendentes da filial 01/01
U_CRMSELLER({"01","01","PRIMEIRA"})

// Carga diária - período específico 
U_CRMSELLER({"03","02","DIARIA"},"","20260301","20260310")

// Automático - usa configuração padrão
U_CRMSELLER()

Tipos de Processamento​

TipoDescriçãoPeríodo Padrão
PRIMEIRACarga inicial completaÚltimos 6 meses
DIARIAProcessamento incrementalOntem

2. U_CRMSELDB(cEmpresaa)​

Função de debug para processar todas as filiais de uma empresa em paralelo.

// Debug empresa 01 - todas as filiais em paralelo
U_CRMSELDB("01")

Características​

  • Processamento paralelo via StartJob
  • Delay controlado (3s entre disparos)
  • Logs detalhados de progresso
  • Ideal para testes e cargas massivas

3. U_CRMSIMPLES(aParam, cTipo, cDataIni, cDataFim)​

Versão alternativa para compatibilidade e testes.

U_CRMSIMPLES({"01","01","PRIMEIRA"})

Configuração​

2. Controle de Execução​

  • ES_SELLULT_03_02: Última execução filial 03/02
  • ES_EMPRCRM: Empresas ativas ("01,03,04,06,11,24,27")
  • ES_FILCRM: Filiais ativas ("01,02")

Endpoints API​

Envio de Notas​

POST /PRTNotaVenda/grupo/{idGrupo}/empresa/{idEmpresa}/CreateNotaVenda
Content-Type: application/json
Authorization: Basic {token}

Finalização Pós-Integração​

POST /PRTNotaVenda/grupo/{idGrupo}/PostAtualizaNotaVenda
Content-Type: application/json
Authorization: Basic {token}

Campos Calculados​

tipoOperacao (V/D)​

CASE 
  WHEN UPPER(SF4.F4_XRESMOV) LIKE '%DEVVEN%' THEN 'D' 
  WHEN UPPER(SF4.F4_XRESMOV) LIKE '%VENDA%' THEN 'V' 
  ELSE 'V' 
END

tipoPagamento (V/P)​

  • "V": À vista (1 parcela ou parcelas vazias)
  • "P": Parcelado (> 1 parcela)

Tabela ZRM - Controle de Integração​

Campos Principais​

CampoTipoDescriçãoExemplo

Status de Operação​

CódigoDescriçãoAção
AAdicionouNova nota inserida
UUpdateNota já existia
XErroFalha na integração

Query Principal​

Tabelas Utilizadas​

FROM SF2010 SF2                    -- Cabeçalho das notas
INNER JOIN SD2010 SD2              -- Itens das notas  
INNER JOIN SB1040 SB1              -- Produtos
INNER JOIN SA1080 SA1              -- Clientes
INNER JOIN SF4210 SF4              -- TES
LEFT JOIN SA3010 SA3               -- Vendedores
LEFT JOIN SE4A01 SE4               -- Condições pagamento
LEFT JOIN ZRM010 ZRM               -- Controle integração

Filtros Aplicados​

-- Filtros principais
SF2.F2_FILIAL = '01'                    -- Filial específica
SD2.D2_FILIAL = '01'                    -- Isolamento por filial
SF2.F2_EMISSAO BETWEEN inicio AND fim   -- Período
SF2.F2_XTPPED != '36'                   -- Exclusões específicas
SB1.B1_TIPO = 'ME'                     -- Apenas mercadorias
D2_XEMPFIL = '.'                        -- Filtro empresa
ZRM.ZRM_CODIGO IS NULL                  -- Não integrados

Operações Incluídas​

-- TES válidas
(SF4.F4_XRESMOV LIKE '%VEND%' OR SF4.F4_XRESMOV LIKE '%DEV%')

Funções de Apoio​

CrmGetEmps(): Lista de Empresas​

aEmpresas := CrmGetEmps()
// Retorna: {"01","03","04","06","11","24","27"}

CrmGetFils(cEmp): Filiais por Empresa​

aFiliais := CrmGetFils("01") 
// Retorna: {"01","02","03"} - filiais da empresa 01

CrmRetFil(cEmp): Detalhes das Filiais​

Consulta tabela SZP para informações completas.

Troubleshooting​

Problemas Comuns​

1. "Nenhuma filial encontrada"​

// Verificar configuração
aFiliais := CrmGetFils("01")
If Len(aFiliais) == 0
    // Problema: ES_FILCRM não configurado ou SZP vazia
EndIf

2. "Erro ao obter configurações"​

// Verificar credenciais
oConfig := U_crmCredl("01")
If oConfig == Nil
    // Problema: crmCredencial.prw não configurado
EndIf

3. "Query não retorna dados"​

  • Verificar filtros por filial (F2_FILIAL, D2_FILIAL)
  • Conferir período de datas (F2_EMISSAO)
  • Validar TES (F4_XRESMOV com VEND/DEV)

4. Status HTTP diferentes de 200​

StatusSignificadoAção
409Nota já existeNormal - registro como "U"
500Erro servidorVerificar API/dados
401Não autorizadoVerificar token
404Endpoint não encontradoVerificar URL

Checklist de Implementação​

Pré-requisitos​

  • Tabela ZRM criada e configurada
  • crmCredencial.prw configurado com URLs e tokens
  • Parâmetros ESSELLULT, ES_EMPRCRM, ES_FILCRM criados
  • Tabela SZP (filiais) populada (opcional)

Testes Básicos​

// 1. Testar configuração
oConfig := U_crmCredl("01")

// 2. Testar busca de filiais  
aFiliais := CrmGetFils("01")

// 3. Testar integração individual
U_CRMSELLER({"01","01","PRIMEIRA"},"","20260301","20260301")

// 4. Testar debug paralelo
U_CRMSELDB("01")

// 5. Verificar logs
U_CRMSTAT()

Validação de Dados​

  • Query retorna notas do período correto
  • Campos tipoOperacao calculados (V/D)
  • Filtros por filial funcionando
  • ZRM registrando integrações
  • API respondendo com status 200/409
